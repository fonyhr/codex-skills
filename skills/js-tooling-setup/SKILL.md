# JS tooling setup

## Process
- Identify the stack (node version, package manager, framework, bundler).
- Propose the minimal change and a clear goal.
- For config changes, mention impact on repo and team (format, lint, build).

## Caution
- Do not change config without known CI impact.
- For version migrations, state compatible ranges.

## Language
- Respond in Czech per the `czech-communication-standard` skill.
