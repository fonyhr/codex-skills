# JS requesting code review

## What to provide to the reviewer
- Short goal of the change (1-2 sentences).
- Key files and areas.
- How to test.
- Risks or open questions.

## Style
- Short, scannable bullets.
- No unnecessary marketing.

## Language
- Respond in Czech per the `czech-communication-standard` skill.
