# Test-driven development

## Process
- First describe expected behavior.
- Write a failing test (minimal case).
- Implement the smallest code change.
- Refactor after the test passes.

## Additions
- Prefer fast unit tests over end-to-end when appropriate.
- For UI tests, emphasize stable selectors.
- For async tests, use explicit waits and timeouts.
